dimensional modeling 
modeling technique 
we store data in form of facts and dimensions 

fact table - ex: stores only numeric values like product id , customer id ,bill,price 
dimensions : diffrnt tables each one is categorized by clusters of data like customers,products 

two types of data models 
star schema  -- mostly used in real world
snowflake schema -- hierarchy of dimesions(not used ) 

if we change dat in source needs to change in dimension tables 

slowly changing dimensions 

type-0  -- assume the dim table doesn't change
type 1 (UPSERT) - update + insert -- overwrite 
changes gets overwrite and updates in fcats table also 

type 2  (HISTORY)
store the history like which data and what data changed 

type 3 (PREVIOUS VALUE)
it will store the previous value for updated value

Medallion Architecture 
it is a data design pattern used to oraganize data in lake house 

1)Bronze layer - push data from source to this layer as it is - no change then
2) Silver layer - processing like cleaning,transformation
3) Gold layer - data stored in form of dimesions and facts, aggregate tables etc as per reqs

we do apply this architecture in cloud data engineer architecture
